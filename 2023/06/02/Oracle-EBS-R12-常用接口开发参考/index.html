AP

仅支持CREDIT/STANDARD，AP

Invoices应付发票【Open Interface】

快速参考

仅支持CREDIT/STANDARD，金额小于零用CREDIT，大于零用STANDARD。REPAYMENT/DEBIT/EXPENSE REPORT/MIXED不支持，可先用CREDIT或者STANDARD，进去之后再Update正式表ap_invoices_all。

参考点 内容
功能导航 N: AP/Invoice
可执行并发程序 APXIIMPT
接口表 ap_invoices_interface、ap_invoice_lines_interface
API ap_import_invoices_pkg
错误信息表 ap_interface_rejections
接口Form N/A
接口Report N/A
Debug Profile 请求参数控制：Trace、Debug、Summary报表
参考 ap_import_invoices_pkg.import_invoices

ap_invoices_interface

字段 说明
org_id 经营单位ID
invoice_id 来自ap_invoices_interface_s
invoice_num 发票号
invoice_type_lookup_code STANDARD是标准发票，CREDIT是Credit Memo
invoice_date 发票日期
vendor_id 供应商ID
vendor_site_id 供应商地点
invoice_amount 发票金额
invoice_currency_code 发票币种
exchange_xxx 如果是外币，需要输入汇率类型、日期、汇率
SOURCE 来源，来自快速编码SOURCE
group_id 组ID，可直接用invoice_id

invoice_line_id 来自ap_invoice_lines_interface_s
accounting_date 财务日期
line_number 行号
line_type_lookup_code 行类型
amount 金额
dist_code_combination_id 账户ID

PO

PO Requisitions 请购单【Open Interface】

快速参考

Oracle自身也通过该接口处理集成问题：MRP生成请购单，内部销售订单生成请购单，库存计划生成请购单等。外部系统通常也需要用该接口先生成请购单后再创建采购订单。

参考点 内容
功能导航 N: PO/Requisitions/Requisitions
并发请求 N: View/Requests/Requisition Import导入新请购单 N: View/Requests/Reschedule Requisitions更新数量和时间
接口表 po_requisitions_interface_all po_req_dist_interface_all po_reschedule_interface
API
错误信息表 po_interface_errors
详细例子 oracle_iface_scripts_pkg.po_req_iface
开发参考
接口表、错误信息表

Table List:

接口表 数据表 说明
po_requisitions_interface_all po_requisition_headers_all po_requisition_lines_all po_req_distributions_all 接口表以line为单位，header和distribution也含在这里面
po_req_dist_interface_all po_req_distributions_all 仅当1行有多个具体分配行时使用该接口；Oracle自身的系统11i之后已经不用了
po_reschedule_interface po_requisition_headers_all po_requisition_lines_all po_req_distributions_all 重新计划，其实就是修改数量和日期，要跑Reschedule Requisitions请求
